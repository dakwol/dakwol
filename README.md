## Архитектурный подход

```txt
src/
├── app/
├── pages/
├── widgets/
├── features/
├── entities/
└── shared/
```

<table>
<tr>
<td width="50%" valign="top">

### Принципы

* Feature-Sliced Design;
* явные типы вместо магии;
* маленькие компоненты;
* разделение UI / model / lib / config;
* декларативная бизнес-логика;
* предсказуемое управление состоянием;
* изоляция фич;
* производительность до абстракций.

</td>
<td width="50%" valign="top">

### Что ценю в коде

* понятность;
* расширяемость;
* строгую типизацию;
* минимальный coupling;
* отсутствие ninja-code;
* читаемую архитектуру;
* стабильную производительность;
* хороший DX.

</td>

## Architecture Style

```txt
src/
├── app/
├── pages/
├── widgets/
├── features/
├── entities/
└── shared/
```

<table>
<tr>
<td width="50%" valign="top">

### Principles

* Feature-Sliced Design;
* explicit types over magic;
* small components;
* UI / model / lib / config separation;
* declarative business logic;
* predictable state management;
* isolated features;
* performance before abstractions.

</td>
<td width="50%" valign="top">

### Code Values

* readability;
* scalability;
* strict typing;
* minimal coupling;
* no ninja-code;
* clean architecture;
* stable performance;
* good DX.

</td>
</tr>
</table>
